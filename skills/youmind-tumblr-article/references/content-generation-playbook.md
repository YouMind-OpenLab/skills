# Content Generation Playbook — Tumblr

Use this when the user gives a topic, angle, or rough idea.

## Step 1 — Choose the right post type

Ask one question internally first:

**What is the actual object here?**

- If it is an argument, lesson, build log, or reflection → text post
- If it is a single strong visual with commentary → photo post

Do not default to text post just because the model can write more words.

## Step 2 — Find the Tumblr angle

Start from reblog value, not search value.

Good Tumblr angles:

- a build lesson with one sharp opinion
- a creator note attached to a concrete artifact
- a behind-the-scenes explanation of one decision
- a reflection with one practical takeaway

## Step 3 — Write the opening for the dashboard

The first paragraph should do one of:

- state the surprising thesis
- surface the problem in plain language
- contrast what people expect vs what actually happened

Avoid:

- “In today’s rapidly evolving landscape...”
- “This article explores...”
- any intro that could fit ten other platforms

## Step 4 — Shape the body

For text posts, a strong default is:

1. Hook
2. Short context
3. 2-5 scannable sections
4. One clean ending line

For photo posts:

1. One-sentence scene setting
2. Why this image matters
3. One insight or invitation

## Step 5 — Add Tumblr-native signals

- 3-8 useful tags
- one memorable line
- one concrete example
- one human choice in phrasing

## Step 6 — Pre-publish checks

- does the first screen earn expansion?
- is the post type correct?
- would this invite a reply or reblog, not just a passive view?
- is the image actually doing work, if present?
