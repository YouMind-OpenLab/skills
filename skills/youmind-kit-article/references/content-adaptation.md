# Kit Adaptation Rules

- Treat Kit as `broadcast + creator profile + public newsletter feed`, not as a plain blog.
- Separate the `subject line`, `preview text`, and `body opening`; each surface does different work.
- Choose `public` or `private` intentionally before drafting the close and CTA.
- Use one clear title/subject field as the H1; do not repeat it inside the body.
- Prefer shorter paragraphs and fewer sections than you would in a long evergreen blog post.
- The first screen should explain what changed and why it matters immediately.
- Keep CTA and cross-sell language restrained and singular.
- If public feed visibility matters, think about thumbnail and creator-profile presentation too.
- Avoid complicated embeds unless the user explicitly asks for them.
- Private drafts should explicitly remind the user they can inspect them in `https://app.kit.com/campaigns`.
