# WeChat Professional Playbook

> Rules card, not a case-study archive.
> Use it when the article needs stronger column logic, first-screen design, or review discipline.

> This file turns case studies and ecosystem research into execution rules for the skill.
> Treat it as the "how strong accounts actually operate" layer, distinct from hard platform constraints.

---

## The System View

For a professional公众号, the article is not a standalone asset. It sits inside a system:

`title -> first screen -> body rhythm -> trust -> action -> follow-up asset -> repeatable column`

If any one link is weak, the piece may still read well but will underperform operationally.

### The 4 working principles

1. **One account = one durable reader promise**
   The reader should know what they reliably get from this account.
2. **One article = one primary reader job**
   Solve one main question, tension, or task. Do not attempt to serve every intent.
3. **One piece = one primary conversion goal**
   Save, comment, next read, follow, add WeCom, join group, open mini-program: pick one primary path.
4. **Series beat isolated hits**
   Repeatable columns outperform random topic spikes over time because they build reader expectation and editorial memory.

## What Strong公众号 Do Consistently

### 1. Promise clearly in the title

The title should usually communicate:

- who this is for
- what concrete value the reader gets
- one tension, contradiction, or surprising angle

Practical default:

`reader / scenario + payoff + tension`

Examples:

- `给第一次带团队的人：3 个你以为是管理，其实是在制造沉默的动作`
- `做 AI 工具的人都在追功能，但真正拉开差距的是这条很土的运营线`

### 2. Win the first screen

The first 120-200 Chinese characters should answer:

- who this article is really for
- what specific gain the reader gets
- why it matters now

Do not spend the first screen on:

- generic background
- scene-setting with no payoff
- throat-clearing like "随着……发展"

### 3. Keep mobile rhythm visible

Professional accounts write for thumb-scrolling, not desktop immersion.

- prefer 2-4 short lines per paragraph
- change pace every 2-4 screens
- use subheads, bullets, tables, quote blocks, charts, or images to reset attention
- put one idea per paragraph whenever possible

### 4. Design the ending for the next step

The ending is not a moral summary. It is a handoff.

Prefer one explicit next step:

- save this
- reply/comment with a specific answer
- read the next article in the series
- follow for the next installment
- open the mini-program / add WeCom / get the resource

Do not pile multiple unrelated asks into the ending.

### 5. Build columns, not only posts

Columnization makes a公众号 feel dependable.

Good column signals:

- fixed topic cluster
- recurring format
- stable publishing rhythm
- recognizable voice and layout
- clear "this belongs with previous pieces" internal linking

## Account Archetypes

Use these archetypes when deciding voice, structure, and CTA.

### Brand / service account

Best when it feels:

- consistent
- useful
- trustworthy
- operationally helpful

Priorities:

- clear structure
- proof and use cases
- light but controlled personality
- strong landing / service handoff

### Expert / operator personal account

Best when it feels:

- experience-backed
- opinionated
- specific
- conversational without being loose

Priorities:

- sharp thesis
- concrete examples
- strong first-person pattern library
- one memorable line readers want to quote

### Companion / media account

Best when it feels:

- ritualized
- emotionally stable
- easy to continue reading

Priorities:

- predictable publishing habit
- strong emotional pacing
- series memory
- recommendation to "read the next one"

## Article Planning Defaults

Before writing, define these 5 items explicitly:

1. `reader_job`
   What job is the reader hiring this article to do?
2. `share_reason`
   Why would someone forward this to one specific person?
3. `primary_conversion_goal`
   What is the one main action after reading?
4. `series_position`
   Is this a standalone piece, part 1, mid-series piece, or recap?
5. `proof_payload`
   What evidence, case, or material makes this piece harder to ignore?

If any one of these is blank, the article is under-specified.

## Process Metrics That Matter

Do not review a piece using only read count.

Track at least:

- open rate / title performance
- first-screen drop-off or early bounce proxy
- read depth / completion proxy
- save / like / comment / share mix
- primary conversion completion

Interpretation rules:

- high opens + weak completion: promise strong, delivery weak
- weak opens + strong completion: content good, packaging weak
- strong completion + weak conversion: handoff weak or CTA mismatched
- strong conversion + weak follow retention: piece converts, but account promise is unstable

## Professional Anti-Patterns

- **The isolated hit:** a good standalone article that does not connect to a column, topic cluster, or next action
- **The beautiful wall:** polished text with weak scanning structure on mobile
- **The multi-CTA dump:** asking for follow, share, comment, add WeCom, and buy in one ending
- **The vague brand voice:** style changes wildly article to article
- **The pseudo-insight title:** tension in the title, but only summary in the body
- **The metrics vanity trap:** optimizing only for total reads while ignoring read depth and conversion

## Case Anchors

Use these as calibration anchors, not imitation targets:

- `深圳卫健委`: high-trust service content with controlled internet fluency
- `丁香医生`: expertise + structure + visual explanation + trust
- `十点读书`: ritualized publishing and emotional companionship
- `日食记`: strong aesthetic system + story-led brand memory

The point is not to mimic their voice. The point is to understand how stable promise, recognizable structure, and repeatable columns create professional strength.
